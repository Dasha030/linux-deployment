# linux-deployment

# Лабораторна робота: Автоматизація розгортання вебзастосунків в ОС Linux

Цей репозиторій містить результати виконання комплексної лабораторної роботи з налаштування безпечного, автоматизованого середовища для вебзастосунку в Linux.

## Індивідуальний варіант

Згідно з номером у списку групи (**N = 27**), було розраховано наступні параметри конфігурації:
- **Спосіб конфігурації ($V_2 = 2$):** Через конфігураційний файл за шляхом `/etc/mywebapp/config.json`.
- **База даних ($V_2 = 2$):** **PostgreSQL** (доступ обмежено лише локальним хостом `127.0.0.1`).
- **Тематика вебзастосунку ($V_3 = 1$):** **Notes Service** (Сервіс текстових нотаток).
- **Порт застосунку ($V_5 = 3$):** **3000** (вебсервер слухає виключно `127.0.0.1:3000`).

---

## Архітектура системи

Система побудована за трирівневою класичною схемою безпечного розгортання:
1. **Nginx (Reverse Proxy):** Приймає зовнішні HTTP запити на порті `80`. Забезпечує маршрутизацію бізнес-логіки та блокує доступ до службових ендпоінтів моніторингу з зовнішньої мережі.
2. **Node.js Application:** Працює у фоновому режимі під керуванням **Systemd** від імені системного користувача `app` з мінімальними привілеями.
3. **PostgreSQL Database:** Надійно ізольована СУБД, яка зберігає дані нотаток.

---

## Реалізовані ендпоінти

### Службові (доступні лише локально):
- `GET /health/alive` — Перевірка працездатності рантайму сервера (повертає `200 OK`).
- `GET /health/ready` — Динамічна перевірка підключення до бази даних PostgreSQL.

### Бізнес-логіка (доступні ззовні):
- `GET /` — Головна сторінка зі списком усіх доступних роутів бізнес-логіки (формат `text/html`).
- `GET /notes` — Повертає список існуючих нотаток (`id`, `title`).
- `POST /notes` — Створення нової нотатки (приймає JSON з полями `title`, `content`).
- `GET /notes/<id>` — Повертає повний вміст конкретної нотатки за її унікальним ідентифікатором.

### Гнучкість форматів (Обробка заголовка `Accept`)
- Якщо клієнт надсилає запит із заголовком `Accept: application/json` (або за замовчуванням) — сервіс повертає структуровані дані у форматі JSON.
- Якщо клієнт запитує `Accept: text/html` — сервер динамічно генерує чисту HTML сторінку без стилів, відображаючи списки у вигляді тегів `<table>`.

## Налаштування безпеки та користувачів

У системі створено чітку ієрархію користувачів для розподілу обов'язків:
1. **`student`** — Власник середовища розробки. В домашній директорії створено унікальний ідентифікаційний файл `gradebook` з обмеженими правами читання (`-rw-------`).
2. **`teacher`** — Користувач з повними правами адміністратора (`sudo`). Пароль: `1245678`.
3. **`operator`** — Користувач технічної підтримки. Завдяки налаштуванням у `/etc/sudoers.d/operator` має обмежене право виконувати команди керування сервісами (`systemctl`) для `mywebapp` та `nginx` без введення пароля адміністратора.
4. **`app`** — Системний бездисковий користувач (`/bin/false`) для ізольованого запуску коду процесу Node.js.
